                                Common Validation and Server Errors
                            
    
                            Fast recovery tips
                        
    
                        
    
                            
    
                                400 Validation failed
                                
    
                                    Payload does not meet field rules
                                
    
                                
    
                                    Example:
                                    POST /api/anime
                                    with a title under 2 characters, a missing
                                    image path, or a year outside the allowed
                                    range.
                                
    
                                
    
                                    Fix: use the form hints, keep required
                                    fields filled, and make sure
                                    img_name, year, genre, synopsis, studio, and
                                    episodes match the limits.
                                
    
                            
    

                            
    
                                400 Invalid JSON
                                
    
                                    Request body cannot be parsed
                                
    
                                
    
                                    Example: sending a body with trailing
                                    commas, unmatched braces, or plain text to a
                                    JSON endpoint.
                                
    
                                
    
                                    Fix: resend valid JSON, or use the sample
                                    body from Endpoint Explorer before trying
                                    the request again.
                                
    
                            
    

                            
    
                                404 Anime not found
                                
    
                                    The record ID does not exist
                                
    
                                
    
                                    Example:
                                    PUT /api/anime/:id
                                    or
                                    DELETE /api/anime/:id
                                    using an ID that was already removed or
                                    never existed.
                                
    
                                
    
                                    Fix: choose an ID from
                                    /api/anime
                                    or refresh the list before retrying the
                                    action.
                                
    
                            
    

                            
    
                                409 Duplicate record
                                
    
                                    The anime already exists
                                
    
                                
    
                                    Example: creating a record with a title or
                                    slug that matches an existing anime entry.
                                
    
                                
    
                                    Fix: adjust the title or slug, then submit
                                    again after confirming the record is not
                                    already in the catalog.
                                
    
                            
    

                            
    
                                500 Server error
                                
    
                                    Backend could not finish the request
                                
    
                                
    
                                    Example: a data file cannot be read, the
                                    backend is offline, or feedback storage
                                    fails during save.
                                
    
                                
    
                                    Fix: confirm the server is running, retry
                                    once, then check the response details or
                                    server logs for the failing subsystem.
